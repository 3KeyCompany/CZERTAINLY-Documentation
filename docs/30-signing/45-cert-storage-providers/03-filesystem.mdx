# File System Certificate Storage Provider

The implementation class of the File System Certificate Storage Provider is:

```
com.czertainly.trident.provider.certificate.FileSystemCertificateStorage
```

The following properties can be configured for the File System Certificate Storage Provider:

export const data = [
    {
        property: "CERT_STORAGE_BASE_DIR",
        description:
            (
                <>
                    Absolute path on the file system, where the certificates are store as files.
                    The file name must follow this rule: <b>[userId]_[keyId]</b>.
                    Certificates are Base64 encoded.
                </>
            ),
        default:
            (
                <>
                    <span class="badge badge--secondary">NONE</span>
                </>
            ),
        mandatory:
            (
                <>
                    <span class="badge badge--success">YES</span>
                </>
            )
    }
];

export const GenTable = ({data}) => (
    <React.Fragment>
        {data.map((dat, i) =>{
            return(
                <tr key={i}>
                    <td><b>{dat.property}</b></td>
                    <td>{dat.description}</td>
                    <td>{dat.default}</td>
                    <td>{dat.mandatory}</td>
                </tr>)
        })}
    </React.Fragment>
);

<table>
    <th>Property</th>
    <th>Description</th>
    <th>Default Value</th>
    <th>Mandatory</th>
    <tbody>
    <GenTable data={data}/>
    </tbody>
</table>

:::caution
The system user running the SignServer must have RW permission to the `CERT_STORAGE_BASE_DIR`.
:::
